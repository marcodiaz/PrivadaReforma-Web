1) Stable core workflows (ordered by delivery priority)

1. User Provisioning

Route: /admin/users

Administrative onboarding surface for role-based account creation/invitation. Enforces role/unit constraints and determines downstream authorization boundaries for all app areas.

2. Visitor QR Issuance

Route: /app/visits

Primary access-control workflow. Creates visitor passes with validity model, usage limits, and person limits. Produces a deterministic display code for guard-side validation and auditability.

3. QR Distribution Modal

From: /app/visits

Operational handoff UI containing scannable payload, token, and copy/delete controls. Supports controlled revocation by allowing pass deletion from the same context.

4. Pets Community Module

Route: /app/pets

Neighborhood content module for pet identity and coexistence data. Includes publication metadata and moderation entry points, reducing low-context community conflicts.

5. Marketplace

Route: /app/marketplace

P2P listing flow with structured listing attributes (price/condition/media/contact). Integrates reporting controls to feed moderation pipelines and maintain trust/safety.

6. Resident Package Flow

Route: /app/packages

Secure package lifecycle visibility: stored -> ready_for_pickup -> delivered. Aligns resident confirmation with guard delivery transition to prevent misdelivery.

7. Incident Prioritization Feed

Route: /app/incidents

Community incident board with score-based prioritization using plus/minus voting. Drives triage focus by social urgency and declared priority.

8. Incident Creation Modal

Overlay on: /app/incidents

Structured input point for title, description, category, and severity. Ensures incidents enter operational queues with normalized fields.

9. Parking Enforcement Reports

Route: /app/parking

Resident-to-guard escalation for assigned parking violations. Maintains status history for accountability and closure traceability.

10. Profile and Credential Hygiene

Route: /app/profile

User identity maintenance for display name and password update. Supports account-level security hygiene and profile consistency.

11. Targeted Push Operations

Route: /admin/push

Manual push dispatch by user/unit/role/global scope. Enables rapid operational broadcast with explicit targeting semantics.

2) Admin/foundation modules (less mature or supervisory)

12. Moderation Reports

Route: /admin/reports

Review/resolve console for reported incident, pet, and marketplace content. Current screenshot displays empty state; action paths activate when open reports exist.

13. Global Package Supervision

Route: /admin/packages

Read-oriented cross-unit package monitoring and filterable state aggregation for operations oversight.

14. Resident Finance Surface

Route: /app/finance

Foundational finance shell intended for statement, fee, and transparency expansion.

15. Debt Management Surface

Route: /admin/debts

Collections-oriented financial placeholder for delinquency control workflows.

16. Admin Finance Surface

Route: /admin/finance

Executive finance shell for budget, inflow/outflow visibility, and governance reporting evolution.

17. Export Surface

Route: /admin/exports

Placeholder for CSV/PDF report generation pipelines for operational and board reporting.

3) Native app advantages (guard-first operations)

Why install the native app instead of relying only on mobile web:

  • Higher reliability for background/locked-screen critical push delivery.
  • Severity-specific vibration/audio channels (SOS vs parking vs incident).
  • Live SLA timers per ticket with breach warnings and escalation hooks.
  • Permission-based location context for dispatch and evidence workflows.
